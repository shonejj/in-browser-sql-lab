PROBLEM IDENTIFIED
─────────────────────────────────────────────────────────────────────────────
✗ Backend container crashes when uploading files >100MB
✗ Entire files loaded into RAM before processing (3x memory bloat)
✗ No memory limits on Docker containers
✗ No file size validation on endpoints
✗ Synchronous processing blocks concurrent uploads
✗ No error handling or temp file cleanup

ROOT CAUSES
─────────────────────────────────────────────────────────────────────────────
1. FastAPI endpoint: content = await file.read()
   → Loads entire file into memory at once
   
2. Docker compose: No memory/CPU limits
   → Container can consume all available RAM
   
3. Nginx config: client_max_body_size 500M
   → Allows very large uploads without backend limits
   
4. No file size validation
   → Oversized files reach backend and crash it

SOLUTION IMPLEMENTED
─────────────────────────────────────────────────────────────────────────────

1. DOCKER RESOURCE LIMITS ✅
   File: docker-compose.yml
   Changes:
   - deploy.resources.limits.memory: 1G (hard limit)
   - deploy.resources.limits.cpus: '2' (CPU limit)
   - deploy.resources.reservations.memory: 512M (guaranteed)
   - healthcheck: Every 30 seconds
   - restart: on-failure

2. NGINX CONFIGURATION ✅
   File: nginx.conf
   Changes:
   - client_max_body_size: 500M → 50M (aligned with backend limit)
   - proxy_request_buffering: off (disable memory buffering)
   - proxy_read_timeout: 300s, proxy_connect_timeout: 30s
   - proxy_send_timeout: 300s

3. BACKEND FILE SIZE VALIDATION ✅
   File: backend/main.py
   Changes:
   - MAX_IMPORT_FILE_SIZE=52428800 (50MB)
   - MAX_MINIO_FILE_SIZE=524288000 (500MB)
   - CHUNK_SIZE=5242880 (5MB per chunk)
   - Added size checks before streaming
   - Returns HTTP 413 for oversized files

4. STREAMING IMPLEMENTATION ✅
   File: backend/main.py
   New function: save_uploaded_file_streaming()
   Changes:
   - Read file in 5MB chunks
   - Write directly to disk (no RAM buffering)
   - Check size during streaming
   - Cleanup temp files on error
   
   Updated endpoints:
   - /api/import (50MB limit)
   - /api/files/upload (500MB limit)
